## The Lead

Thursday, late in the UTC day. Claude Code v2.1.163 landed at 21:52 UTC with the feature that managed-fleet operators have been waiting for since dynamic workflows shipped: version pinning. Two new settings, `requiredMinimumVersion` and `requiredMaximumVersion`, let organizations specify an allowed version range. Claude Code refuses to start outside that range, no override. For teams running Claude Code across hundreds of developer machines or in CI, this closes a real gap. The rest of the release is polish and bug fixes: a persistent Bazel/EDR bash failure, a `claude -p` hang on backgrounded-command-final-exit, and background agent sessions that now auto-update without a cold restart.

The version just before it, v2.1.162 (June 3, 21:31 UTC, 3.5 hours before this window opened), added `waitingFor` to `claude agents --json`. That field tells automation what a blocked session is actually waiting on. Combined with today's version pinning, the week's Claude Code arc is clear: the team is building for fleets, not just individual developers.

## Release Log

### Claude Code

**2026-06-04 — Claude Code v2.1.163**
`[CODE]`
Source: https://github.com/anthropics/claude-code/releases

Shipped June 4 at 21:52 UTC. Key additions: `requiredMinimumVersion` and `requiredMaximumVersion` as managed settings, blocking startup outside the allowed range. `/plugin list` command with `--enabled` and `--disabled` filters. A "c to copy" shortcut inside `/btw` that puts raw markdown on the clipboard. Hooks gain `hookSpecificOutput.additionalContext` on `Stop` and `SubagentStop`, allowing a hook to push feedback back into the turn rather than just terminating it. Skills get `\$` escape syntax for literal dollar signs before digits in command bodies. `stdio` MCP servers now receive `CLAUDE_CODE_SESSION_ID` on `--resume`.

Bug fixes: `claude -p` no longer hangs after final result when backgrounded commands are slow to exit. Bash failures under Bazel and EDR-protected workflows are resolved. Background agent sessions now auto-update to new Claude Code versions without requiring a cold restart. Multiple terminal rendering and session-handling fixes included.

How to use: `claude update` or reinstall. Managed version settings deploy via `managedSettings.json`.

**2026-06-03 — Claude Code v2.1.162 (3.5 hours before window; noted for completeness)**
`[CODE]`
Source: https://github.com/anthropics/claude-code/releases

Shipped June 3 at 21:31 UTC. Added `waitingFor` field to `claude agents --json`, showing what a waiting session is blocked on. Explicit `Grep` and `Glob` tool listings now work on native builds with embedded search. `/effort` confirms when a chosen level will persist as the default for new sessions. Slash commands in the autocomplete menu now fill into the prompt instead of executing immediately. Remote Control now shows as a persistent footer pill rather than a startup message. Renamed Windsurf to Devin Desktop in menus.

Fixes: silent startup hang when config directory is read-only. WebFetch permission rules not applying to preapproved domains. Windows path matching failures with backslashes and case-variant paths. Interrupt (Esc) at turn start silently dropped in stream-json and SDK sessions. MCP per-server timeout values below 1000ms being floored.
