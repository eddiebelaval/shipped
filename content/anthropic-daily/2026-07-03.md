Shipped. Daily, Thursday, July 3, 2026

Three Claude Code releases tighten the permission model and patch background agents; both SDKs add a preview header for agent memory, landing July 22.

## Lead

Claude Code v2.1.200, shipping July 3 at 16:52 UTC, moves the default permission mode to "Manual" across CLI, VS Code, and JetBrains. Claude now stops and asks before acting, where it previously did not. The companion change lands in the same build: AskUserQuestion dialogs no longer auto-continue. The idle timeout is opt-in, via /config. Eight additional fixes cover background session stability, daemon handover, rate-limited subagents, and tmux 3.4+ rendering.

v2.1.201 followed at 23:50 UTC with a single item: Sonnet 5 sessions no longer inject mid-conversation system role messages for harness reminders. v2.1.199, shipping the previous day at 23:35 UTC, added stacked slash-skill invocations (up to five in one call) and resolved several streaming and background-agent reliability issues.

Both SDKs shipped July 2 with one addition: the `agent-memory-2026-07-22` beta header. The date in that string is July 22. Something is landing in three weeks.
