`[CODE]`

Dynamic workflow size config, two new OpenTelemetry attributes, and 14 bug fixes.

**How to use:** `claude update`. New /config option `workflow.size` accepts `small`, `medium`, or `large` to guide agent-count in workflow runs. Advisory, not a hard cap. Workflow-spawned agents now emit `workflow.run_id` and `workflow.name` in telemetry traces.

**Fixes in this release:** Ctrl+R history search crash on accept or cancel; /rename reversion on job restart; transient mTLS handshake failures during certificate rotation; Remote Control commands failing with "Unknown command" in interactive sessions; images and files from Remote Control app dropped without captions; unreliable clickable sign-in URLs over SSH; chat-opening crash in `claude agents` with "currently running as a background agent" loop; unicode quote corruption in workflow scripts; unbounded voice dictation retry loop on audio failures; wrong permission mode shown in Remote Control sessions; session resume slowness in repos with many git worktrees; installer retry on transient connection drops; skill instruction duplication on re-invoke; MCP error clarity for servers with `url` but no `type`.

**Also:** `/review <pr>` reverts to fast single-pass review. Use `/code-review <level> <pr#>` for multi-agent. Workflows list gets wider titles, a dedicated time column, shorter model names.
