## Lead

Claude Code v2.1.139 landed at 18:43 UTC. Two features anchor it. Agent View (Research Preview) surfaces every session, running, blocked, or completed, in a single list accessible via `claude agents`. The `/goal` command lets you set a completion condition once; Claude keeps working across turns until it hits it, with a live overlay showing elapsed time, turns, and tokens. Forty-plus bug fixes ride along, including a deadlock fix for expired credentials and a hard 16 MB cap on previously unbounded memory growth in HTTP/SSE MCP servers. ([release notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.139))

## Release Log

### CODE

#### 2026-05-11 — [Claude Code v2.1.139](https://github.com/anthropics/claude-code/releases/tag/v2.1.139)
`[CODE]`

Agent View (Research Preview): `claude agents` lists all running, blocked, and completed sessions. `/goal` sets a multi-turn completion condition with a live timer overlay. `/scroll-speed` tunes scroll behavior with live preview. `claude plugin details <name>` shows component inventory and projected per-session token cost. Transcript navigation: `?` for shortcuts, `{`/`}` to jump between user prompts, `v` to toggle the shortcut panel. Hook additions: `args: string[]` field for exec-form commands, `continueOnBlock` option on `PostToolUse` hooks. MCP: stdio servers now receive `CLAUDE_PROJECT_DIR`, and `/mcp Reconnect` picks up `.mcp.json` edits without a restart. 40+ bug fixes, including a deadlock on expired credentials, an `autoAllowBashIfSandboxed` shell-expansion gap, the MCP memory cap (16 MB per frame), and mouse scroll in Cursor and VS Code 1.92 through 1.104.

**How to use:** `claude update` or reinstall. Run `claude agents` to open Agent View.
