**RELEASE LOG**

**C. Claude Code**

#### 2026-05-07 — [Claude Code v2.1.133](https://github.com/anthropics/claude-code/releases)
`[CODE]`

Version 2.1.133 adds a `worktree.baseRef` setting (`fresh` | `head`) controlling whether new worktrees branch from `origin/<default>` or local `HEAD`, new `sandbox.bwrapPath` and `sandbox.socatPath` managed settings for Linux/WSL, and a `parentSettingsBehavior` admin-tier key for SDK `managedSettings` policy merge control. Hooks now receive the active effort level via `effort.level` in the event JSON and the `$CLAUDE_EFFORT` environment variable. Memory usage improved under pressure. Bug fixes: parallel session credential failures after refresh-token race, `Edit`/`Write` permission rules incorrectly matching drive roots, Remote Control stop not fully canceling sessions, `/effort` changing effort level across concurrent sessions, and subagents failing to discover project and plugin skills.

**How to use:** Update via `claude update` or reinstall. The `worktree.baseRef` setting lives in project `.claude/settings.json`. Hooks integrations pick up `$CLAUDE_EFFORT` automatically.

**E. Agent SDKs**

#### 2026-05-07 — [anthropic-sdk-typescript v0.95.1](https://github.com/anthropics/anthropic-sdk-typescript/releases)
`[SDK-TS]`

Redacts API key values from debug log output. Previously, API keys could appear in plaintext in debug logs. A problem in any environment where logs ship to a shared sink.

**How to use:** `npm install @anthropic-ai/sdk@0.95.1` or `pnpm update @anthropic-ai/sdk`.
