**The security fix that sat twelve bullets deep**

Claude Code 2.1.145 shipped at 21:31 UTC and fixed a permission-prompt bypass: bare variable assignments to non-allowlisted environment variables in Bash commands were being auto-approved. In agentic workflows, that is a real security hole, not a cosmetic patch. The fix sits at bullet twelve of a nineteen-item changelog, which is the right amount of understatement. ### C. Claude Code

#### 2026-05-19 — Claude Code v2.1.145 (https://github.com/anthropics/claude-code/releases/tag/v2.1.145)
`[CODE]`

Scripting, observability, plugin discoverability, and a security fix. `claude agents --json` surfaces live sessions as JSON for external tooling. OTEL gains `agent_id` and `parent_agent_id` on `claude_code.tool` spans with corrected trace parenting for background subagents. Status line JSON now includes GitHub repo and PR context. `/plugin` Discover and Browse show a plugin's full manifest before install. `claude agents` tab title shows awaiting-input count. Bash fix: bare variable assignments to non-allowlisted env vars were auto-approved, now blocked. Also fixed: MCP error messaging for missing required arguments, spinner freeze after terminal resize, Windows PowerShell `;` separator, voice push-to-talk in agent view, task list ordering on concurrent creation, Agent Teams non-ASCII name header encoding, `/review` GraphQL regression on repos with Classic Projects, skill `context: fork` infinite loop, Read tool returning hard error instead of partial content on token-limit overflows.
