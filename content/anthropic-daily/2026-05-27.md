## Release Log

### CODE

#### 2026-05-27 - Claude Code v2.1.152 ([github.com/anthropics/claude-code/releases](https://github.com/anthropics/claude-code/releases))
`[CODE]`

`/code-review --fix` applies review findings to the working tree. `/simplify` aliases to it. Skills and commands can declare `disallowed-tools` in frontmatter to remove tools while active. `/reload-skills` added. `SessionStart` hooks can return `reloadSkills: true` or set session title via `hookSpecificOutput.sessionTitle`. New `MessageDisplay` hook event for transforming assistant message text before render. `pluginSuggestionMarketplaces` managed setting for allowlisting plugin sources. Auto mode no longer requires opt-in consent. Vim mode: `/` in NORMAL opens reverse history search. `/usage` breakdown includes large session files. Thinking summaries render as markdown, capped at 10 lines (Ctrl+O for full).

#### 2026-05-28 (00:52 UTC, in window) - Claude Code v2.1.153 ([github.com/anthropics/claude-code/releases](https://github.com/anthropics/claude-code/releases))
`[CODE]`

Adds `skipLfs` option to `github`/`git` plugin marketplace sources to skip Git LFS downloads. Status line commands receive `COLUMNS` and `LINES` env vars. `claude agents` autocomplete now suggests native slash commands and bundled skills. PR column shows `PR #N` (single) or `N PRs` (multiple). `claude doctor` shows results of last update attempt; one-time notice when npm global install cannot auto-update, with `/doctor` listing fixes. MCP servers and connectors auth notifications combined into a single message. macOS: background agents appear as "Claude Code" in Privacy and Security with persistent permissions. `/model` saves selection as default (press `s` for current session only). Bug fixes: stateful MCP server reconnect-looping, custom API gateway OAuth credential handling, subagent MCP servers ignoring `--strict-mcp-config` and `--bare`, Windows PowerShell installer false success, memory usage on resume with many stored sessions, CLI exit in stream-json mode with closed stdin, malformed `file://` links.
