## Lead: Sonnet 5

Claude Sonnet 5 launched June 30 and immediately became the default model for every Claude Free and Pro user. Not a preview. Not a limited rollout. Default, as of today.

The positioning is explicit: Sonnet 5 is an agentic model. It can plan, use tools like browsers and terminals, and run autonomously at a level that until recently required Opus-class compute and Opus-class pricing. On agentic benchmarks it runs close to Opus 4.8. The context window is one million tokens, native.

Pricing is the other story. Through August 31, 2026: $2 per million input tokens, $10 per million output tokens. After that, standard pricing at $3/$15. That introductory window is real; if you're building long-running agents and haven't recalculated, now is the time.

Available today: Claude API (model ID `claude-sonnet-5`), claude.ai (new default for Free and Pro), Claude Code (update to v2.1.197), Amazon Bedrock, Google Cloud, and Microsoft Foundry. The system card published the same day.

#### 2026-06-29: Claude Apps Gateway for Amazon Bedrock and Google Cloud ([announcement](https://claude.com/blog/introducing-the-claude-apps-gateway), [docs](https://code.claude.com/docs/en/claude-apps-gateway))
`[CODE]`

A self-hosted control plane for Claude Code deployments on enterprise cloud infrastructure. Single stateless container on Linux, backed by PostgreSQL. Provides: corporate SSO, centrally enforced policy, role-based access, per-user cost attribution, and spend limits (daily/weekly/monthly per org, group, or user).
