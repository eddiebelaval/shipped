### v2.1.199 — the release where failures stopped lying

Claude Code v2.1.199 shipped tonight at 23:35 UTC ([releases](https://github.com/anthropics/claude-code/releases)) with two dozen changes, and the headline feature is small: stacked slash-skill invocations, so `/skill-a /skill-b do XYZ` now loads all leading skills, up to five, where it previously loaded only the first and silently dropped the rest.

The real theme runs through the fixes. Streaming responses hit by a mid-stream server error used to discard the partial output; now the partial is kept, flagged incomplete. Subagents cut off by a rate limit used to fail silently; now they return their partial work to the parent. Subagents that hit an API error used to report it as a *successful result*; now the error surfaces. SSL certificate failures used to burn through retries before showing guidance; now they fail immediately with the fix hint. Hooks that exited with an error used to hide their stderr; now it lands in the transcript. Five separate fixes, one posture: the agent stack stops swallowing its own failures.

The SSL fix deserves its own blast radius. The failure lived on TLS-inspecting corporate proxies and machines missing `NODE_EXTRA_CA_CERTS` — which is to say, on enterprise laptops, exactly where the person hitting it has the least ability to diagnose a tool that spins through retries before admitting the certificate chain is the problem. Failing fast with the fix hint turns a ticket into a one-line config change.

That last class is the one that costs people real hours. A subagent that errors and reports success is the silent-corruption bug of agent orchestration — the parent proceeds on an empty result, the final output is subtly wrong, and nothing in the transcript says why. If your multi-agent runs have produced inexplicably hollow results lately, this is a plausible culprit, and it was fixed tonight.

Also in the release: the Linux background-agent daemon no longer kills itself — and every running agent — after an unclean shutdown leaves a corrupted worker record. **The move:** update before your next long orchestrated run, and start stacking skills.
