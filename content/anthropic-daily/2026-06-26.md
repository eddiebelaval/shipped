## The Dig: Claude Code v2.1.195

The day's other item is a [Claude Code point release](https://github.com/anthropics/claude-code/releases), and it is a maintenance drop — which, by this desk's standing rule, is a day to dig rather than demote. Three fixes carry it.

**The mechanism.** Hook matchers with hyphenated identifiers now use exact matching instead of substring matching. That is a small line with a real failure mode behind it: under substring matching, a hook keyed to `pre-commit` could also fire on `pre-commit-ci` or any identifier that merely contained the string — silent, wrong triggering that is miserable to debug because nothing errors. Exact matching closes it. Separately, a new `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` environment variable disables click-and-drag in fullscreen mode while preserving wheel scroll — the escape hatch for terminal setups where mouse capture fought the user's own tooling.

**The blast radius.** The voice-dictation repair is the one that cost people time. macOS dictation is fixed, including auto-submit for languages without spaces — Japanese, Chinese, Thai — where the previous logic waited for a word break that never came, so dictated input just sat there. Anyone dictating in a spaceless language had a feature that silently didn't work; now it does. Voice mode on Linux improved in the same release, and Remote session startup adds a provisioning checklist, so a session that fails to come up tells you why instead of hanging.

**The read.** None of this is a headline, and that is the tell of a tool that has graduated from feature-chasing to reliability. The hook-matcher and dictation fixes are the kind of bug you only bother closing when the surface is mature enough that the long tail is what's left. A point release that spends itself on exact-match semantics and spaceless-language dictation is a product being maintained for people who depend on it daily.

**The builder's move.** `claude update`, or reinstall from the GitHub releases page. If you script hooks with hyphenated names, this release changes their matching behavior — check that your matchers still fire where you expect.
