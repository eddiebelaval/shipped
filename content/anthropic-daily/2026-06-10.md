## The Dig

### Managed Agents grew up: cron and credential vaults

The agent platform got the two things that separate a demo from production: it can run on its own, and it can hold a secret safely. **Cron scheduling** (public beta) lets you attach an agent to a schedule on the Claude Platform and have it fire automatically — no external trigger, no babysitting a cron box of your own. Add `cron_schedule` to the agent definition and it runs.

The quieter, better half is the **credential vault**. Secrets stop living in the system prompt and start living at the network boundary: keys bind only on outbound requests to domains *you've* allowlisted, and never enter model context at all. That's the difference between "the model could in principle leak my API key in a completion" and "the model never sees the key." For anyone who's been white-knuckling secrets in prompt text to make an agent useful, this is the pass that lets you stop. No separate scheduler fee — it bills through existing Claude Platform usage. **Move:** put creds in the vault, set the allowlist, schedule the agent, delete the key from your prompt.

### Claude Code: sub-agents that spawn sub-agents

Two drops in 24 hours. **v2.1.170** (Jun 9) put Fable 5 inside Claude Code and fixed a real papercut — sessions launched from the VS Code integrated terminal (or any shell with inherited Claude Code env vars) weren't saving transcripts, so work was vanishing for the IDE crowd specifically. **v2.1.172** (Jun 10) is the structural one: **sub-agents can now spawn their own sub-agents, up to five levels deep.** That turns a flat fan-out into an actual tree — an orchestrator can delegate to delegators. Powerful, and worth a word of caution: five levels of recursion is five levels of token spend and five levels of things to debug, so reach for the depth only when the work genuinely nests. Also in 172: Bedrock now reads your region from `~/.aws` when `AWS_REGION` is unset (drop the env var if your config's already there), a plugin-marketplace search bar, a `model` attribute on the `claude_code.lines_of_code.count` OTEL metric, and fixes across background sub-agents, the model picker, permission rules, WebFetch wildcard domains, and idle CPU. **Move:** `claude update`; nesting is on by default.
