## The Dig

### Managed Agents grew up: a scheduler and a vault

A demo becomes production the day it can run on its own and hold a secret without bleeding it. Managed Agents (public beta) got both at once. **Cron scheduling** lets you bind an agent to a schedule on the Claude Platform and have it fire on its own — no external trigger, no cron box of your own to babysit. Add `cron_schedule` to the agent definition; it runs.

The quieter half is the better half. The **credential vault** moves secrets out of the system prompt and down to the network boundary: a key binds only on outbound requests to a domain *you* allowlisted, and never enters model context at all. That is the gap between "the model could leak my API key in a completion" and "the model never sees the key." Anyone who's been white-knuckling secrets in prompt text just to make an agent do real work can stop. And the pattern is the tell — a scheduler plus a credential boundary plus no separate scheduler fee is Anthropic quietly building the spine of an always-on agent runtime, not shipping two convenience features. It bills through existing Claude Platform usage. **Move:** put creds in the vault, set the allowlist, schedule the agent, delete the key from your prompt.

### Claude Code: sub-agents that spawn sub-agents

Two drops in twenty-four hours. **v2.1.170** (Jun 9) put Fable 5 inside Claude Code and fixed a real papercut: sessions launched from the VS Code integrated terminal — or any shell with inherited Claude Code env vars — weren't saving transcripts, so work was quietly evaporating for the IDE crowd specifically. **v2.1.172** (Jun 10) is the structural one. **Sub-agents can now spawn their own sub-agents, up to five levels deep.** That turns a flat fan-out into a real tree: an orchestrator can delegate to delegators. Powerful, and worth one word of restraint — five levels of recursion is five levels of token spend and five layers to debug, so reach for the depth only when the work genuinely nests. Also in 172: Bedrock reads your region from `~/.aws` when `AWS_REGION` is unset (drop the env var if your config already has it), a plugin-marketplace search bar, a `model` attribute on the `claude_code.lines_of_code.count` OTEL metric, and fixes across background sub-agents, the model picker, permission rules, WebFetch wildcard domains, and idle CPU, plus performance work for long conversations. **Move:** `claude update`; nesting is on by default.
