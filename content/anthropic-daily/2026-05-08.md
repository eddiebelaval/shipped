#### 2026-05-08 — [Claude Code v2.1.136](https://github.com/anthropics/claude-code/releases)
`[CODE]`

Dense fix release with two new capability additions. New: `settings.autoMode.hard_deny` lets teams define auto-mode classifier rules that block unconditionally regardless of user intent or configured allow-exceptions; `CLAUDE_CODE_ENABLE_FEEDBACK_SURVEY_FOR_OTEL` re-enables the session quality survey for enterprises capturing responses via OpenTelemetry.

Fix cluster: MCP servers configured in `.mcp.json`, plugins, and claude.ai connectors silently dropping after `/clear`; a login loop from concurrent credential writes overwriting freshly-rotated OAuth tokens; MCP OAuth refresh token races across multiple concurrent servers; a 400 API error when extended thinking emitted a redacted block after a tool call; `--resume` and `--continue` failing to find sessions when the project path contains underscores; plan mode failing to block file writes when a matching `Edit(...)` allow rule exists. WSL2: image paste from the Windows clipboard now works via PowerShell fallback.

**How to use:** `claude update` or reinstall. Add `settings.autoMode.hard_deny` rules to `.claude/settings.json` for unconditional classifier blocks in auto mode.
