## Lead

**Claude Code v2.1.147: 28 fixes and a renamed command**

The most visible change in [v2.1.147](https://github.com/anthropics/claude-code/releases/tag/v2.1.147) is a rename: `/simplify` is now `/code-review`, and it ships with effort levels (`/code-review high`) and a `--comment` flag that posts findings directly as inline PR comments on GitHub. Clean integration, no clipboard archaeology.

The second change worth tracking is operational: pinned background sessions (`Ctrl+T` in `claude agents`) now survive idle, restart in place when an update lands, and are shed under memory pressure only after non-pinned sessions clear first. Sessions that hold their state through an update cycle are sessions you can rely on.

The rest is a 28-item fix list covering PowerShell on Windows, MCP server pagination, auto-updater retries, diff rendering performance, and edge cases in plugin and shell snapshot handling. Nothing changes what the tool can do. Several things change how reliably it does it.

## Release Log

### C. Claude Code

#### 2026-05-21 - Claude Code v2.1.147 ([GitHub releases](https://github.com/anthropics/claude-code/releases/tag/v2.1.147))
`[CODE]`

`/simplify` renamed to `/code-review` with effort-level reporting (`/code-review high`) and a `--comment` flag for GitHub PR inline comments. Pinned background sessions now survive idle and restart in place on updates. 28 additional fixes: auto-updater retries, diff rendering performance, enterprise login enforcement, PowerShell on Windows with winget/Microsoft Store installations, MCP server pagination (resources, templates, prompts past page 1 were being dropped), shell snapshot user-function handling, plugin agent multi-type declarations, hook `if` condition matching, and more.

**How to use:** `claude update` to get v2.1.147. Run `/code-review` in any session, or `/code-review high` for a more thorough pass. Add `--comment` to post results as inline GitHub PR review comments.
