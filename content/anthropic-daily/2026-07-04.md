## Lead

Claude Code v2.1.201 shipped at 23:50 ET on July 3, which is 03:50 UTC on July 4, making it the only confirmed release Anthropic produced on the holiday. The change is one line in the release notes: Claude Sonnet 5 sessions in Claude Code no longer use the mid-conversation system role to deliver harness reminders.

That sounds like internal plumbing. It is. Here is what it means.

The Claude Code harness is the scaffolding that wraps Claude's model capabilities into a coding agent. It manages context windows, enforces tool permissions, tracks session state, and periodically injects reminders about the active rules of engagement. Those reminders are operational scaffolding, not user instructions. With Sonnet 5's initial integration into Claude Code, those reminders were being delivered via the system role mid-conversation, not just in the initial system prompt at session start, the way you would expect, but intermittently, as the session ran.

The system role is supposed to be immutable framing. The contract, from the model's perspective, is that the system turn establishes the ground truth of the conversation and stays fixed. Injecting additional system-role messages mid-session breaks that contract. It forces the model to decide whether a new system-role message is a superseding instruction, a reminder of existing rules, or noise. Sonnet 5 is a more agentic model than its predecessors, and more agentic models are more sensitive to instruction ambiguity in long contexts. The patch resolves the ambiguity by routing harness reminders through a different channel entirely.

The blast radius: anyone running Sonnet 5 sessions in Claude Code on complex or long-running workflows. If you noticed Sonnet 5 behaving inconsistently mid-session, weighting a reminder as a new instruction and shifting behavior without an obvious user trigger, this was likely the cause. The symptom would have been subtle and difficult to attribute: the model doing something reasonable, just not what you expected at that point in the session.

**How to use:** Update Claude Code (`claude update`). The fix applies automatically to all new Sonnet 5 sessions. No configuration change needed.
