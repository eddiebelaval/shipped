## Release Log

### C. Claude Code

#### 2026-06-12: Claude Code v2.1.175 ([CHANGELOG](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md))
`[CODE]`

Added `enforceAvailableModels` managed setting: when enabled, the `availableModels` allowlist also governs which model Default resolves to, giving organizations tighter control over which models org members can access.

**How to use:** Set `enforceAvailableModels: true` in your managed settings. Default model selection will be constrained to your `availableModels` allowlist. Update via `claude update`.

---

#### 2026-06-11: Claude Code v2.1.173 ([CHANGELOG](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md))
`[CODE]`

Fixed Fable 5 model names with a `[1m]` suffix not being normalized. Fable 5 ships with 1M context by default, so the suffix is now stripped automatically. Fixed a spurious "sandbox dependencies missing" startup warning on Windows when sandbox was enabled in settings.

---

#### 2026-06-11: Claude Code v2.1.172 ([CHANGELOG](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md))
`[CODE]`

Sub-agents can now spawn their own sub-agents, up to 5 levels deep. Amazon Bedrock reads the AWS region from `~/.aws` config files when `AWS_REGION` is not set, matching AWS SDK precedence. Added a search bar when browsing marketplace plugins in `/plugin`. Fixed: sessions using 1M context without usage credits getting permanently stuck; JetBrains terminal flicker via synchronized output; Shift+non-ASCII drops in Kitty keyboard protocol terminals; PowerShell command validation hang on Windows.

**How to use:** Update via `claude update`. Nested sub-agents are on by default. Bedrock region resolution requires no config change.
