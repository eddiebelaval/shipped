Claude in Microsoft Foundry: generally available
Claude is generally available in Microsoft Azure Foundry as of today, running on NVIDIA GB300 Blackwell Ultra GPUs. Claude Opus 4.8 and Claude Haiku 4.5 are in the Messages API with prompt caching and extended thinking included. Azure-native authentication, billing, and governance are built in. An optional US data zone is available for teams with data-residency requirements.
How to use: Access through your Azure subscription. See platform.claude.com/docs/en/build-with-claude/claude-in-microsoft-foundry for setup.

CODE
Claude Code v2.1.196
Broad reliability and quality drop. Org admins can now set default models in the console, surfaced as "Org default" or "Role default" in the model picker. Sessions get readable names on start. File attachments are clickable in chat (Cmd/Ctrl-click opens in Finder or Explorer). MCP security tightened: self-approved repo servers no longer auto-spawn in untrusted workspaces, which now show a pending-approval badge. Streaming idle watchdog on by default, aborting and retrying if a stream produces no events for five minutes. Per-frame render work cut by roughly 37%.
Key fixes: Background session crash when transcript probe misread files. Rate-limit warning flickering and over-counted telemetry under parallel requests. Duplicate recap lines after background sessions. MCP OAuth scope requests on GitLab self-hosted and other enterprise IdPs now request only required scopes. PowerShell git diff/git grep and quoted patterns with | no longer false-positive on exit 1. Keyboard focus stuck when opening the claude agents panel. Mid-turn crash recovery for Remote sessions. /context showing 0 tokens on Bedrock. Set CLAUDE_ENABLE_STREAM_WATCHDOG=0 to disable the new watchdog.
