Four Claude Code patch releases tightened enterprise model governance and unlocked five-deep sub-agent nesting, while DXC deployed Fable 5 as the default model inside its managed services platform for banks, airlines, and government agencies.
Lead Story
DXC goes all in

DXC Technology, the managed services firm running back-office infrastructure for banks, airlines, insurers, and government agencies, announced a multi-year global alliance with Anthropic on June 11. DXC tested Claude across its 115,000-person workforce in 70 countries before rolling it to clients. Claude wrote more than 95% of the code for DXC OASIS, its new AI-native orchestration platform, with Fable 5 as the default foundation model.

DXC is training tens of thousands of Claude-certified forward-deployed engineers to install Claude inside the systems regulated industries have trusted for decades. This is not a new product launch. This is an established firm putting Claude at the center of what it sells.

Claude Code v2.1.175
Added enforceAvailableModels managed setting. When enabled, the availableModels allowlist also governs which model Default resolves to, giving organizations tighter control over model access across the org.
How to use Set enforceAvailableModels: true in your managed settings. The Default model will be constrained to models in your availableModels allowlist. Update via claude update.

Claude Code v2.1.174
Added wheelScrollAccelerationEnabled setting to disable mouse-wheel scroll acceleration in fullscreen mode. Fixed the /model picker hiding the model family that Default resolves to.
Jun 11
CODE
Claude Code v2.1.173
Fixed Fable 5 model names with a [1m] suffix not being normalized. Fable 5 ships with 1M context by default, so the suffix is now stripped automatically. Fixed a spurious "sandbox dependencies missing" startup warning on Windows when sandbox was enabled in settings.
Jun 11
CODE
Claude Code v2.1.172
Sub-agents can now spawn their own sub-agents, up to 5 levels deep. Amazon Bedrock reads the AWS region from ~/.aws config files when AWS_REGION is not set, matching AWS SDK precedence. Added a search bar when browsing marketplace plugins in /plugin. Fixed: sessions using 1M context without usage credits getting permanently stuck; JetBrains terminal flicker via synchronized output; Shift+non-ASCII drops in Kitty keyboard protocol terminals; PowerShell command validation hang on Windows.
How to use Update via claude update. Nested sub-agents are on by default. Bedrock region resolution requires no config change, just ensure your ~/.aws/config has the region set.
