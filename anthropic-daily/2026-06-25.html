Anthropic accused Alibaba of extracting Claude capabilities via fraudulent accounts; Claude Code v2.1.193 shipped the same evening.

Anthropic formally accused Alibaba's Qwen AI lab of running a large-scale campaign to extract Claude's capabilities in software engineering and agentic reasoning, using thousands of fraudulent accounts. The accusations arrived in a letter to US senators and White House officials, first reported by CNBC on June 24 and followed by Bloomberg on June 25.

Alibaba ADRs fell to a 16-month low in Hong Kong following the disclosure. Anthropic has not published the letter publicly.

Separately, Claude Code v2.1.193 shipped at 21:45 UTC: auto-mode routing broadened, an OpenTelemetry response event added, and three backgrounding bugs fixed that had been leaving phantom agents in sessions.

Claude Code v2.1.193
Auto-mode gets a new autoMode.classifyAllShell setting that routes all Bash and PowerShell commands through the classifier, not just arbitrary-code-execution patterns. A new claude_code.assistant_response OpenTelemetry log event emits model response text as a structured log, redacted by default and opt-in via OTEL_LOG_ASSISTANT_RESPONSES=1. Live file path autocomplete lands in bash mode (! prefix). Idle background shell commands now auto-reap under memory pressure (disable with CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1). Three backgrounding bugs fixed: spurious cancellation warnings on exit, phantom "general-purpose (resumed)" subagent spawned on main turn backgrounding, and pinned agents being re-prompted after auto-update.
How to use Run claude update or reinstall. Set autoMode.classifyAllShell: true in your Claude settings to route all shell commands through the auto-mode classifier. Set OTEL_LOG_ASSISTANT_RESPONSES=1 in your environment to enable response logging for your observability pipeline.
