### Apps

#### 2026-06-18: Claude Code Artifacts: live, shareable HTML pages from agent sessions
`[APPS]`

Claude Code can now surface its work as an interactive, shareable web page, live-updating as the agent and connected data sources change. Teams can use them for PR walkthroughs, incident dashboards, internal tools, and checklists. Pages are private to the org. Version history is in beta. Restricted to Team and Enterprise accounts.

**How to use:** Available in Claude Code for Team and Enterprise subscribers. No additional configuration required. Start a Claude Code session and look for the Artifacts option to surface the session as a shareable URL.

**Why it matters:** Turns an ephemeral terminal session into a persistent, shareable record. The terminal is private by default; this makes it collaborative on demand.

#### 2026-06-18: Enterprise-Managed Authorization for MCP connectors, with Okta
`[APPS]`

Admins can now provision MCP server access through their identity provider once, and employees inherit that access on first login with no per-app OAuth prompts. The feature works across Claude, Claude Code, and Cowork for Team and Enterprise accounts. Seven MCP providers support the standard at launch: Asana, Atlassian, Canva, Figma, Granola, Linear, and Supabase. Okta is the first supported identity provider.

**How to use:** Configure your identity provider (Okta) in Claude admin settings. Provision MCP connectors once. Users inherit access on first login. Beta for Team and Enterprise. See the MCP blog post for full setup details.

**Why it matters:** Per-user OAuth prompts were the largest friction point blocking enterprise-scale MCP adoption. Centralized provisioning removes that entirely.

Source: https://blog.modelcontextprotocol.io/posts/enterprise-managed-auth/

### Claude Code

#### 2026-06-19: Claude Code v2.1.183: auto mode safety, deprecation warnings, attribution settings
`[CODE]`

Auto mode now blocks destructive git commands (git reset --hard, git checkout -- ., git clean -fd, git stash drop) unless you explicitly asked to discard local work. git commit --amend is blocked for commits the agent did not make this session. Terraform, Pulumi, and CDK destroy commands require explicit request per stack. Model deprecation warnings added in print mode (-p) and for models set in agent frontmatter. New attribution.sessionUrl setting omits the claude.ai session link from commits and PRs in web and Remote Control sessions.

**How to use:** Update Claude Code (claude update or reinstall). New safety gates are automatic in auto mode. Run /config --help to list available shorthand config keys.

Additional fixes: thinking block display errors, WebSearch in subagents, terminal cursor positioning, fullscreen corruption on Windows Terminal. Background task and webhook trigger handling fixes. Subagent tmux pane and focus mode improvements.

Source: https://code.claude.com/docs/en/changelog
