Claude Design grows up

Anthropic shipped a significant Claude Design update on June 17, adding the features enterprise design teams had been waiting for. Design systems can now import from GitHub repositories, design files, or raw uploads, so generated prototypes match what your brand team mandated. The previous version lacked consistency when applying those systems. That problem now has a name and a fix, rather than a workaround.

Bidirectional sync with Claude Code via /design-sync and /design commands handles the handoff. A designer can start from a local codebase, so generated assets already contain elements from your production products. Direct canvas editing closes the gap with traditional tools. Export connectors expanded to Adobe, Canva, and Miro. An admin role locks company components so individual contributors cannot drift off-brand. Available to Claude Pro, Max, Team, and Enterprise customers. Source: claude.com/blog.

Claude Design: design system imports, code sync, canvas editing
Claude Design now imports design systems from GitHub repositories, design files, or raw uploads. Bidirectional sync with Claude Code via /design-sync and /design commands covers the design-to-code handoff. Direct canvas editing, export connectors to Adobe, Canva, and Miro, and an admin role that locks company components ship in the same update. Available to Claude Pro, Max, Team, and Enterprise customers.
How to useOpen Claude Design from your Claude account. Use /design-sync to push to Claude Code; use /design to start from a running codebase.
