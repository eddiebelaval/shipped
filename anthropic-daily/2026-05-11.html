Three releases. Agent View lands in Claude Code, AWS client support ships across both SDKs, and the rest of the board stays quiet.

Lead

Agent View is the kind of feature that sounds obvious once it exists. Before today, running multiple Claude Code sessions meant context-switching blind: no unified view of what was running, what was blocked, what had finished. Version 2.1.139, released Monday afternoon, changes that. Run claude agents from any terminal and you get a single list of every session, organized by state.

The research preview ships alongside a /goal command that lets Claude continue working across turns until a defined completion condition is met, and a continueOnBlock hook option for PostToolUse that gives operators finer control over automated runs. Both features push in the same direction: multi-session orchestration, handled natively in the CLI.

Release Log
C. Claude Code
2026-05-11 CODE Claude Code v2.1.139

Agent View (research preview) adds claude agents, a single-list view of all Claude Code sessions (running, blocked on you, done). New /goal command sets a completion condition and Claude works across turns until it is met, in interactive, -p, and Remote Control modes, with a live overlay showing elapsed time, turns, and tokens. /scroll-speed tunes mouse wheel scroll speed with live preview. claude plugin details <name> shows plugin component inventory and projected per-session token cost.

Transcript navigation: ? for keyboard shortcuts, {/} to jump between user prompts, v to toggle the shortcut panel. Hook improvements: args: string[] exec-form field for direct command spawning without shell; continueOnBlock config option for PostToolUse. MCP: stdio servers now receive CLAUDE_PROJECT_DIR env var; /mcp Reconnect picks up .mcp.json edits without restart. Remote Control subagent requests carry x-claude-code-agent-id and x-claude-code-parent-agent-id headers. Settings: Remote Control, /schedule, and claude.ai MCP connectors disabled when ANTHROPIC_API_KEY is set. Numerous bug fixes across terminal rendering, MCP servers, permissions, and UI.

How to use: claude update or reinstall. Then run claude agents to open Agent View.
