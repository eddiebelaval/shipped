One Claude Code release in the window: v2.1.163 ships managed version pinning, Bazel and EDR bash fixes, and a handful of hook and plugin improvements.

The Lead

Thursday, late in the UTC day. Claude Code v2.1.163 landed at 21:52 UTC with the feature managed-fleet operators have been waiting for since dynamic workflows shipped: version pinning. Two new settings, requiredMinimumVersion and requiredMaximumVersion, let organizations specify an allowed version range. Claude Code refuses to start outside that range. No override. For teams running Claude Code across hundreds of developer machines or in CI, this closes a real gap. The rest of the release is polish and fixes: a persistent Bazel and EDR bash failure, a claude -p hang on backgrounded-command-final-exit, and background agent sessions that now auto-update without a cold restart.

The version just before it, v2.1.162 (June 3 at 21:31 UTC, 3.5 hours before this window opened), added waitingFor to claude agents --json. That field tells automation what a blocked session is actually waiting on. Combined with today's version pinning, the week's Claude Code arc is clear: the team is building for fleets, not just individual developers.

2026-06-04 CODE
Claude Code v2.1.163

Shipped June 4 at 21:52 UTC. Managed version pinning: requiredMinimumVersion and requiredMaximumVersion settings block startup outside the allowed range. Intended for organizations that need consistent tooling across developer fleets or CI pipelines.

Also ships: /plugin list with --enabled and --disabled filters. A "c to copy" shortcut inside /btw puts raw markdown on the clipboard. Hooks gain hookSpecificOutput.additionalContext on Stop and SubagentStop, so a hook can pass feedback back into the turn instead of just terminating it. Skills get \$ escape syntax for literal dollar signs before digits in command bodies. stdio MCP servers now receive CLAUDE_CODE_SESSION_ID on --resume.

Bug fixes: claude -p no longer hangs after final result when backgrounded commands are slow to exit. Bash failures under Bazel and EDR-protected workflows resolved. Background agent sessions auto-update to new Claude Code versions without requiring a cold restart. Multiple terminal rendering and session-handling fixes included.

How to use: claude update or reinstall. Managed version settings deploy via managedSettings.json.
Just outside the window (noted for completeness): v2.1.162, June 3 at 21:31 UTC, added waitingFor to claude agents --json (shows what blocks a waiting session), fixed slash-command autocomplete filling vs. executing immediately, introduced Remote Control as a persistent footer pill, and resolved a silent startup hang on read-only config directories.
